Move to staging server to another Amazon EC2 instance type

Because dev and staging runned at different instance types (I see difference in available memory at least), please move staging at same instance type like dev uses. They will allow us to analyze possible problems with load at system.

Please notify everyone at #hah-dev, #hah-qa and @rami-alameddine especially about maintenance timeframe, that should take up to 1h. Rami will ping client what env will be unavailable and tests should be stopped at their side.

Then:

  1. Create staging instance image in AWS EC2 console.
  2. Run instance image at another type of instance.
  3. Check what all opened ports are same for instance.
  4. Switch DNS to new IP.
  5. Stop old instance (not terminate that!)
  6. Week later if everything is ok - terminate old staging instance.
